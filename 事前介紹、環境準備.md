## GitHub Copilot 是什麼

GitHub(微軟)開發的AI程式碼開發工具
主要有三個使用方式

1. Visual Studio 2022(最新版本內建)
	- 可以自動補全
	- 僅支援 instructions、prompts、MCP
	- visual studio 獨有功能: @Modernize、 @分析工具...等
	- 建議搭配 Copilot CLI 使用

2. VS Code(延伸模組)
	- 可以自動補全
	- 支援較多的客製化功能
		instructions、prompts、MCP、Agent、Skills、Hooks、Plugins
	- UI介面完善

3. Copilot CLI(終端機工具)
	- 無法補全程式碼
	- 支援較多的客製化功能(同 VS Code)
	- 較豐富的斜線指令
	- 不綁定編輯器

### 額外補充
4. [VS Code - Agent Window](https://code.visualstudio.com/docs/agents/agents-window)
	- 跨專案管理 Agent

5. [GitHub Copilot app](https://github.com/features/ai/github-app)
	- 一樣是跨專案管理，但似乎跟 github 的生態綁定較深

6. [Copilot SDK](https://github.com/github/copilot-sdk)
	- 將 copilot 嵌入到系統中

## 事前準備

### 1. 更新工具
- 更新 **VS Code** 或 **Visual Studio** 到最新版本
- 若 VS Code 沒有出現更新的按鈕，檢查視窗上方"說明" -> "關於" 第一行如果是(user setup)
	1. 不要以系統管理員身份開啟 VS Code，應該就會出現更新的按鈕
	2. 或是考慮移除 VS Code，下載 VS Code 的 System Installer 重新安裝

### 2. 安裝 Copilot
- Visual Studio 2022 的 copilot 是內建的，應該無需另外安裝
- VS Code 確認有安裝 延伸模組 **GitHub Copilot Chat**

### 3. 安裝 Copilot CLI
建議用 winget 安裝，就不用理會不同專案的 node.js 版本：

```powershell
winget install GitHub.Copilot
```

winget 是 win11 內建的，如果 winget 無法被識別，有可能是資料夾路徑沒有被加到 使用者環境變數 Path

用 powershell 執行
```powershell
[Environment]::SetEnvironmentVariable(
  "Path",
  [Environment]::GetEnvironmentVariable("Path", "User") + ";$env:LOCALAPPDATA\Microsoft\WindowsApps",
  "User"
)
```

如果用 npm 安裝：

```bash
npm install -g @github/copilot
```

補充：npm 方式需要 **Node.js 22+**。

### 4. 啟動
```bash
copilot
```
出現介面後輸入`/login`，登入 github 帳號
